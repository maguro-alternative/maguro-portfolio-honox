# ADR-002: プライベート記事を別のプライベートリポジトリで管理する

## ステータス

承認済み

## 背景

プライベート記事を git で管理したい。  
ただし、ポートフォリオの公開リポジトリには含めたくない。

## 決定

プライベート記事専用の別リポジトリ（プライベート）を作成し、  
GitHub Actions 経由で R2 へ同期する。

## 検討した選択肢

### A. 別のプライベートリポジトリ（採用）

- 公開リポジトリに一切含まれない
- git の履歴・差分・ブランチが使える
- GitHub Actions で push 時に自動で R2 へ同期できる

### B. 暗号化して公開リポジトリに含める

- 復号鍵の管理が複雑
- git で差分が見にくくなる
- 暗号化ミスで漏洩するリスクがある

### C. git submodule でプライベートリポジトリを参照

- 公開リポジトリから submodule の存在は見えるが内容は見えない
- submodule の操作が煩雑でミスしやすい
- CI 設定が複雑になる

## 結果

別リポジトリ方式が最もシンプルで安全。  
GitHub Actions のワークフローを公開リポジトリ側に持ち、  
`PRIVATE_REPO_TOKEN` シークレットでプライベートリポジトリを checkout する。  
push のたびに R2 が最新状態に保たれるため、手動操作が不要。
