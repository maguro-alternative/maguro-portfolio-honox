# ADR-001: プライベート記事のストレージに R2 を採用する

## ステータス

承認済み

## 背景

プライベートブログ記事を保存する場所が必要。  
要件は以下の通り。

- GitHub の公開リポジトリに記事の内容が含まれないこと
- 無料枠で運用できること
- HonoX (Cloudflare Workers) から読み取れること

## 決定

Cloudflare R2 にプライベート記事を保存する。

## 検討した選択肢

### A. R2（採用）

- エグレス（転送量）が無料
- Workers から `c.env.R2_BUCKET.get()` で直接読み取れる
- 無料枠: ストレージ 10GB・読み取り 1,000 万回/月

### B. Cloudflare D1

- SQL で管理できるが、Markdown テキストを DB に入れるのは扱いにくい
- スキーマ管理が必要になり、シンプルなブログには過剰

### C. Cloudflare KV

- 結果整合性のため書き込み後すぐ反映されない
- 読み取りは高速だが、プライベート記事は頻繁に更新しないため優位性が薄い

## 結果

R2 はバイナリ・テキストを問わずファイルをそのまま保存できるため、  
Markdown ファイルとの相性が良い。転送量無料も個人サイト運用において決定的な利点。
